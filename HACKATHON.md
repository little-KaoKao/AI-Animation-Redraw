# AI手书复刻 (AI Animation Redraw)

> 上传一段动画视频 + 一张角色图，AI 自动将每一帧重绘为你的角色，保留原始动画的动作、节奏和音频。

---

## 痛点与动机

"手书"（手绘动画）是二次元社区非常流行的创作形式——创作者参考一段已有动画，用自己的原创角色逐帧重绘。但这个过程极其耗时：一段 10 秒的动画可能需要重绘上百帧，即使是熟练画师也需要数小时甚至数天。

**我们的目标：用 AI 将这个过程从数天缩短到数分钟。**

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 智能视频分析 | 自动检测分辨率、帧率、拍数模式（如二拍三拍），去除重复帧 |
| 三视图生成 | 根据单张角色图自动生成正面/侧面/背面参考 |
| 宫格批量重绘 | 将帧组合为宫格图，通过 AI 批量重绘，大幅提升效率 |
| 并发处理 | 最高 20 路并发调用 API，充分利用算力 |
| 单帧重 Roll | 对不满意的宫格可单独重新生成，无需重跑全部流程 |
| 自动合成 | 重绘帧自动按原始时序 + 音频合成为最终视频 |
| 断点续传 | 中断后可从上次完成的阶段恢复，不浪费已完成的工作 |
| 素材缓存复用 | 同一视频/角色在不同项目间复用分析结果和三视图 |

## 处理流程

```
上传视频 + 角色图
    │
    ▼
[1] 视频分析 — 检测分辨率、帧率、拍数模式
    │
    ▼
[2] 提取非重复帧 — 智能去重，减少无效重绘
    │
    ▼
[3] 宫格合成 — 将帧拼为 1/4/9 宫格图
    │
    ▼
[4] 三视图生成 — AI 生成角色正/侧/背参考
    │
    ▼
[5] 批量重绘 — 并发调用 AI 重绘所有宫格
    │          （支持单张重 Roll）
    ▼
[6] 拆分 + 合成视频 — 还原帧序列，叠加原始音频
    │
    ▼
  完成！下载视频
```

## 技术栈

| 层级 | 技术 |

| 语言 | Python 3.12+ |

---

## 亮点与创新

1. **宫格批处理策略** — 将多帧合成为一张宫格图送入 AI，单次 API 调用即可重绘多帧，大幅降低成本和耗时
2. **智能拍数检测** — 自动识别动画的帧保持模式（二拍、三拍等），精准去重，避免冗余计算
3. **断点续传 + 僵尸检测** — 服务重启后自动检测中断的任务，用户可一键恢复
4. **素材级缓存** — 视频分析结果和三视图按素材 ID 缓存，跨项目复用
5. **细粒度控制** — 支持单张宫格重 Roll、暂停/恢复、宫格数可选（1/4/9）

## Demo 演示流程

1. 打开 Web UI → 新建项目
2. 上传一段短动画（3~5 秒效果最佳）+ 一张角色图
3. 选择宫格数 → 点击"开始处理"
4. 观察实时进度：分析 → 提取 → 合成 → 三视图 → 重绘 → 拆分 → 合成视频
5. 预览宫格重绘结果，不满意的可以单独重 Roll
6. 下载最终视频，对比原始动画
